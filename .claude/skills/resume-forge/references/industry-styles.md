# Industry-Specific Style Guide

This reference is loaded only when the user specifies a target industry.

## Technology / Software Engineering

- **Style**: `modern` recommended
- **Font**: Sans-serif (Inter, Calibri, Helvetica)
- **Section order**: Summary → Skills → Experience → Projects → Education
- **Emphasis**: Technical skills first, quantified impact in bullets
- **Tips**: Include GitHub/portfolio links. List technologies per role.
  Projects section is highly valued for early-career candidates.
- **Length**: 1 page for <5 years experience, 2 pages acceptable for senior

## Finance / Banking / Consulting

- **Style**: `classic` recommended
- **Font**: Serif (Garamond, Georgia, Times New Roman)
- **Section order**: Education → Experience → Skills → Certifications
- **Emphasis**: Education first (especially for entry-level). GPA if >3.5.
  Deal sizes, AUM, revenue impact in bullets.
- **Tips**: Conservative formatting. No color. No creative layouts.
  Precise date ranges matter.
- **Length**: Strict 1 page for most levels

## Academia / Research

- **Style**: `classic` recommended (modify to add Publications section)
- **Font**: Serif (Times New Roman, Garamond)
- **Section order**: Education → Research → Publications → Teaching → Skills
- **Emphasis**: Publications list (use citation format for field).
  Research descriptions with methodology and findings.
- **Tips**: CVs (not resumes) are standard — no page limit.
  Include grants, fellowships, conference presentations.
- **Length**: No limit (academic CVs are typically 3-10+ pages)

## Design / Creative

- **Style**: `modern` recommended
- **Font**: Sans-serif with personality (but not too much for ATS)
- **Section order**: Summary → Experience → Projects/Portfolio → Skills → Education
- **Emphasis**: Portfolio link is essential. Visual design of the resume
  itself is evaluated. Project descriptions should focus on process and impact.
- **Tips**: Color accents are expected. Link to portfolio prominently.
  Brief case study descriptions in bullets.
- **Length**: 1 page

## Healthcare / Medical

- **Style**: `classic` recommended
- **Font**: Serif (professional appearance expected)
- **Section order**: Licenses/Certifications → Experience → Education → Skills
- **Emphasis**: License numbers and states. Certifications front and center.
  Patient volume, outcomes, and compliance metrics in bullets.
- **Tips**: Include license/certification expiration dates.
  List EHR systems experience (Epic, Cerner, etc.).
- **Length**: 1-2 pages

## Government / Public Sector

- **Style**: `classic` recommended
- **Font**: Serif (Times New Roman standard)
- **Section order**: Summary → Experience → Education → Certifications → Skills
- **Emphasis**: Security clearance level (if applicable). GS level experience.
  Compliance and regulatory experience.
- **Tips**: Federal resumes follow a different format entirely (USAJobs).
  This guide covers state/local government and government contractors.
- **Length**: 2 pages acceptable, federal can be 4-5+
