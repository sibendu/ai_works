# Essential Claude Code Skills for Business AI Automation

| Skill | Key Function & Benefits | Primary Use Cases |
| :--- | :--- | :--- |
| **1. Skill Creator** | **Automates skill development.** It transforms plain English instructions or SOPs into reliable, packaged Claude skills without requiring manual `.md` file editing. | Converting a real estate agency's manual property description process into a reusable automation. |
| **2. Superpowers** | **Enforces senior developer workflows.** It mandates a "plan-test-review" process in isolated environments, preventing rushed or sloppy code. | Building mission-critical systems, such as HVAC dispatch tools or marketing agency reporting systems. |
| **3. GSD (Get Shit Done)** | **Manages context engineering.** It prevents "context rot" by spawning fresh sub-agents for individual tasks and includes an autonomous mode for planning and execution. | Large-scale projects where requirements might otherwise be dropped or forgotten during long, single sessions. |
| **4. Review Skills (`/re` & `/ultra-review`)** | **Automated quality assurance.** `/re` provides fast local feedback, while `/ultra-review` uses parallel cloud agents to verify bugs and security flaws with high precision. | Running `/re` for continuous feedback and `/ultra-review` before merging critical code like payment systems or database migrations. |
| **5. Context Mode** | **Optimizes the context window.** It filters "garbage" log data to save space and stores session history in a local SQL database to prevent Claude from forgetting tasks. | Deep development sessions (3+ hours) involving heavy data outputs, such as Playwright snapshots or extensive access logs. |
| **6. Claude Mem** | **Persistent long-term memory.** It captures decisions and fixes across sessions using a local vector database, eliminating the need to re-explain project context. | Resuming complex projects after weeks of inactivity without repeating the "startup tax" of caught-up conversations. |

***
